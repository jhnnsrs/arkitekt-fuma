# Architecture

Arkitekt is built as a set of small, largely independent services that can run on a single
machine, a self-managed cluster, or the cloud. This section explains the cross-cutting concerns
that hold those pieces together.

- **[Service Discovery](/docs/design/architecture/service-discovery)** — how apps locate and
  authenticate to the services running in a given deployment via the Fakts protocol.
- **[Security](/docs/design/architecture/security)** — the measures Arkitekt uses to protect
  your data and isolate untrusted plugins, from authentication and sandboxing to TLS.
- **[Provenance & Audit Logging](/docs/design/architecture/provenance)** — how every change to
  your data is recorded with who changed it, under whose authority, and during which task.
