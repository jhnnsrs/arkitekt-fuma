## A note on Workflows

A **Workflow** is a graph of nodes wired together, where the output of one node becomes the input of the next. Each node in the graph runs an [Action](/docs/design/terminology/action). Workflows let you compose individual tasks into a larger, reusable pipeline. How workflows are stored and run is handled by the [Fluss](/docs/design/services/fluss) service, and you can see them in action in the [Showcases](/docs/showcases).
