## What is an action?

Actions are a core conceptual element of the arkitekt platform, that represent the
functionality that Arkitekt can do for you. As such you can think of an action as a provider of a bioimage task.

E.g. an action can be a functionality provided by an app that maximum projects an image, or applies a gaussian filter
to an image, but also a functionality that will ask you the user to select a file, or label an image. An action can
therefore be anything that you can envision as a task.

## Input and Output

Actions in Arkitekt are defined by their input and output. This means that an action can be a simple as an action that
takes an image as an input and outputs an image, or more complex as an action that two images and some information
about the images as an input and outputs a table of results.

## What is an action not?

The most important feature to understand about Arkitekt actions is that they are not bound to an implementation.

This means that an action can be implemented in many different ways **AND**, by various different apps.
For example, here the show Action is implemented by the Fiji app, but also by the Napari app. These apps
can implement the same action in different ways, as long as they respect the input and output and the general
logic of the action. As actions are the primary way to interact with the platform, this means that you can
rest assured that the action will behave the same way, no matter which app implements it, but also that
you can choose the app that you like the most.

<Callout type="info">

Also this abstraction, comes with some additional benefits for the application developer as they can use this
abstraction to automatically test their app against the action specification. More about this in the developer
sections. As apps implementations are the primary way to interact with the platform, this means that even though part
of their code can change each version, but users can still interface with the app in the same way, until the developer
decides to change the input and output of the action.

</Callout>
