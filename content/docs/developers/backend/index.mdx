Arkitekt's [services](/docs/design/services) are Django applications that share a small set of
focused libraries, each owning one concern so the services don't have to reinvent it:

- **[kante](https://github.com/jhnnsrs/kante)** — the GraphQL transport (ASGI) and request
  context the other layers build on.
- **[authentikate](https://github.com/jhnnsrs/authentikate)** — identity: it verifies tokens and
  resolves the acting user, client, and organization for each request.
- **[koherent](/docs/developers/backend/koherent)** — provenance-aware audit logging: it turns
  every data change into a durable, attributed history row.

<Callout type="info">
These libraries are the building blocks behind the [Security](/docs/design/architecture/security)
and [Provenance](/docs/design/architecture/provenance) guarantees described in the design
section. If you want the *concepts* first, start there; if you want to wire them into a service,
the pages here are the hands-on reference.
</Callout>

## Where to start

- [Koherent](/docs/developers/backend/koherent) — add provenance & audit logging to a service.
